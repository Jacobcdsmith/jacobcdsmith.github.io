# ChatGPT Conversation History Analysis

Comprehensive Jupyter notebook suite for analyzing your ChatGPT conversation history.

## Overview

This analysis suite provides deep insights into your ChatGPT usage patterns, topics, and conversation dynamics through 5 comprehensive Jupyter notebooks.

**Your Data:**
- **1,221 conversations** from July 2023 - March 2025
- **23,497 total messages**
- **Average 19.2 messages** per conversation
- **2,600 tool messages** (code execution, browsing, etc.)

## 🚀 Getting Started

### Prerequisites

- Python 3.9 or higher
- Jupyter Notebook or JupyterLab

### Installation

1. **Install required packages:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Ensure data files are in the parent directory:**
   - `../conversations.json` - Your parsed ChatGPT export (or source data file)

3. **Launch Jupyter:**
   ```bash
   jupyter notebook
   ```

4. **Run notebooks in order:**
   - Start with `01_Overview_and_Statistics.ipynb`
   - Each notebook builds on previous ones
   - Notebook 05 requires output from notebooks 01, 03, and 04

### Quick Start

```bash
# From C:\Users\user directory
cd chatgpt_analysis_notebooks

# Install dependencies
pip install -r requirements.txt

# Launch Jupyter
jupyter notebook
```

Then open and run notebooks in order (01 → 02 → 03 → 04 → 05).

### Running the Notebooks

1. **Run in Order:** Notebooks build on each other
   - Notebook 01 creates the base DataFrame
   - Notebook 03 adds topic information
   - Notebook 04 adds complexity and engagement metrics

2. **Cell Execution:** Run all cells in sequence
   - Some calculations depend on previous cells
   - Use "Run All" for complete analysis

   - Close unused notebooks
   - Restart kernel if needed
   - Consider sampling for t-SNE visualization

## 🛠️ Troubleshooting

### Common Issues

**1. Module not found errors:**
```bash
pip install --upgrade -r requirements.txt
```

**2. NLTK data missing:**
The notebooks automatically download required NLTK data, but if issues persist:
```python
import nltk
nltk.download('punkt')
nltk.download('stopwords')
```

**3. Memory errors with t-SNE:**
Notebooks automatically sample for t-SNE, but you can reduce further:
```python
sample_size = min(500, len(combined_texts))  # Reduce from 1000
```

**4. Encoding errors:**
Ensure files are UTF-8:
```python
df = pd.read_csv('file.csv', encoding='utf-8')
```

**5. Date parsing errors:**
Check date format:
```python
df['create_date'] = pd.to_datetime(df['create_date'], errors='coerce')
```
